Canvas allows students to subscribe to a calendar feed with all your classes. When instructors make changes to the calendar, the feed is also updated. With the link to the feed, students can import the Canvas calendar into any calendar app like iCal or Google Calendar. The link to the feed can be found by going to:

Canvas >> Calendar >> Calendar Feed

The problem with this feed is that all courses are combined into one calendar. This means that courses cannot be color coded or categorized within whichever calendar app you use.

There is a solution! The Canvas API allows users to retrieve calendar feeds for each course independently. Students can subscribe to these course specific calendar feeds on a per-course basis, allowing color coding, separation of calendars by account, and categorization of schedules. There is no easy way built into the Canvas UI to get the links to these calendars so that's where this tool comes in. Rather than writing out a set of confusing instructions, I built this walkthrough to help other students find the links to those calendars without needing to really read too much into the details. Go to the walkthrough to give it a try!

There are of course pros and cons of both methods. Either type of calendar feed, composite or course specific, update dynamically when instructors make changes to class schedules. The composite calendar updates when you enroll in entirely new courses whereas the course specific method will need to be used for each course you enroll in. The con of the composite calendar is again that isn't great for organization that the course specific method addresses.
